Instructions from Saturday, July 26, 2014

Instructions to Add Pages

  1. Start in iTerm2
  2. type "git checkout -b add_three_pages" which takes you to a new branch off of the master
  3. go to sublime "sop"
  4. open index.html (note: to get rid of sidebar hit cmd+k cmd+b)
  5. In the header, add an h1 called "Home"
  6. Under the mainContent div, add an h2 called "Welcome to Our Fun Site"
  7. Under the footer, add a p tag with © 2014
  8. change the hrefs to "index.html", "about.html", "contact.html"
  9. in the head, add a stylesheet link "css/style.css"
  10. go back to iTerm2
  11. type "mkdir css" hit return
  12. type "touch css/style.css" hit return
  13. type "sublime css/style.css" hit return
  14. Now you should be in Sublime Text on the css page. Add h1 with a color of red and h2 with a color of blue. (I wrote down the command shortkeys, but it's easier for me to use the arrows)
  15. Save your CSS page cmd+s
  16. Close that file cmd+w
  17. Now you are back on the index.html file in Sublime. Hit cmd+shift+s to save as "about.html" in the same really_fun folder
  18. You'll need to change the h1 and h2 text to "About" and "About Our Fun Site"
  19. Hit cmd+shift+s again to now save as the "contact.html" file
  20. Again, you'll need to change the h1 and h2 text to "Contact" and "Contact Us"
  21. hit cmd+s and switch back to iTerm2
  22. type "git status" hit return. You'll see the files are in red as they've been updated but the updates are not added.
  23. type "git add ." hit return
  24. now type "git status" again and hit return. You can now see the files are in green and everything is working.
  25. To commit to these changes, type "git commit -m "added 3 pages and css" and hit return. The '-m' signifies a 'message' and anything you type in the quotes should be a quick note about the changes you made. Commit often so you have numerous commits with one or two changes each, that way if you need to backtrack you don't have to go back to square one, but just back a couple moves.
  26. type "git status" and hit return. It should say nothing to commit, working directory clean.
  27. type "sop" and hit return to take you back to Sublime Text.
  28. Open your page in the browser. Check to see that it's all working. If something is wrong, you'll have to fix it and save and redo the steps to add and commit the updated files.
  29. Now that it's working, go back to iTerm2
  30. type "git checkout master" and hit return to take you back to the master (you've been working on the branch add_three_pages).
  31. type "git merge add_three_pages" and hit return. Everything will merge.
  32. to publish this to git hub so it can be shared, type "git push" and hit return.
  33. Lastly, go to github.com and refresh your browser to see the pages you added!

Other notes

  • iTerm2 also called "the terminal"
  • to override anything denied actions, use the term "sudo"
  • You can add a file by typing "touch" and "index.html" or whatever you want the file name to be. Make sure you are in the correct folder before doing so.
  • Make a folder with "mkdir" and name
  • "ls" will show you what's listed in a folder
  • In Sublime Text, ctrl+w = wrap. Use the type bar at the bottom to wrap your highlighted area in a tag. Anything highlighted can be indented left or right with cmd+[ or ].
  • sop is an alias to open up sublime text. Another alias is "refrash" for refresh bash.
  • In COMMAND MODE in Sublime, you can move the cursor with the following hot keys:
    • j = down
    • k = up
    • h = left
    • l = right
    • gg = top
    • GG = bottom
    • dd = delete line
